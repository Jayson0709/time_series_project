Summary Page for Different Algorithms


PAA - Piecewise Aggregate Approximation

Features:

  • Efficient, easy to understand.

Disadvantages:

  • Can miss some important patterns in large datasets.
  • Does not have good tightness of lower bound.

Analysis:

  • Time complexity: O(n)
  • Memory requirement: O(n)

OneD_SAX - OneD_Symbolic Aggregate Approximation

Features:

  • Uses precomputed distances (efficient).
  • The distance measure defined on the reduced space lower bounds the distance measure defined on the original space.

Disadvantages:

  • Hard to capture segment trend information.
  • Need to choose average symbol and slope.

Analysis:

  • Time complexity: O(nlog2(n))
  • Memory requirement: O(n^2)

DFT - Discrete Fourier Transformation

Features:

  • A sine-cosine basis space is a very familiar concept. It provides a natural framework for optimizations of transform coder design with inputs where the perceptual effects of signal distortion are best understood in the frequency domain.
  • Can provide accurate results.

Disadvantages:

  • Requires each harmonic to be calculated separately, which requires much more processing power.
  • Does not include location information, only frequency information, and thus has difficulty in representing transients.

Analysis:

  • Time complexity: O(nlog(n))
  • Memory requirement: O(n^2)

DWT - Discrete Wavelet Transformation

Features:

  • Fast and Flexible.
  • Can be used to denoise.

Disadvantages:

  • Takes time to choose a proper wavelet.
  • Computational intensive for fine analysis.

Analysis:

  • Time complexity: O(n)
  • Memory requirement: O(n)

SVD - Singular Value Decomposition

Features:

  • SVD is typically used on sparse data.
  • SVD is commonly used indirectly to explore the main eigenvalues of a transition matrix, estimated from observed trajectories of a certain length.
  • Simplifies data, removes noise, may improve algorithm results.

Disadvantages:

  • SVD of multi-variate time-series is of course very common. SVD cannot meaningfully be directly applied to a uni-variate time-series.
  • Transformed data may be difficult to understand.

Analysis:

  • Time complexity: O(n^3)
  • Memory requirement: O(n^2)

APCA - Adaptive Piecewise Constant Approximation

Features:

  • A raw time series is divided into several segments with unequal lengths based on important points.
  • The observations in each segment are substituted by a mean value.
  • The adaptive piecewise constant approximation is more flexible than the piecewise aggregate approximation.

Disadvantages:

  • Not as fast as PAA or PLA.

Analysis:

  • Time complexity: O(nlog2(n))
  • Memory requirement: O(n)

PLA - Piecewise Linear Approximation

Features:

  • Better tightness of lower bound than PAA.
  • PLA possesses small index dimension and low computational overhead.
  • Moreover, this type of representation is more friendly to human visual experience.

Disadvantages:

  • Takes time to choose the dividing points and the number of segments.
  • Not suitable for discrete data or sparse data.

Analysis:

  • Time complexity: O(n)
  • Memory requirement: O(n)
